# Bot measurement report

Generated by `sim/run.ts`. **5,000 hands per profile**, 100bb deep, blinds 10/20, seed 42, button alternated, bot is seat 0.

> This is a measurement of the bot **as-is**. It reports win-rate and action
> frequencies and flags deviations from exploitatively-correct play. It does
> NOT modify the bot and makes no claim that the bot is "good" or "fixed".

`bb/100` is the bot win-rate (positive = bot winning). The `±` is a 95%
confidence half-width (1.96 × SE) computed from per-hand variance. GTO = bot
with no opponent read; EXPL = bot with opponent tracking / exploit adjuster on.

> Runtime note: the harness DEFAULTS to 100,000 hands/profile (`npm run sim`),
> but the bot runs a Monte-Carlo equity estimate on every decision, so 100k ×
> 4 profiles × 2 modes ≈ 800k bot-hands takes ~1h on this machine. This report
> was generated from the hand count above; the flagged leaks are STABLE across
> 2k / 5k / 20k runs (same leaks, the CIs just tighten), so the conclusions
> hold. Re-run `npm run sim 100000` for the full-resolution numbers.

## Leak classification

- **preflop-range** — the open/defend/3bet/fold-to-3bet frequencies (driven by `src/core/ranges/*`).
- **postflop-heuristic** — cbet / fold-to-cbet / river-bet / showdown behavior (driven by the postflop decision logic in `src/core/engine.ts`).
- **net-distillation** — anything attributable to the distilled policy net path (when the net, not the ranged heuristic, picks the action class).

Note: this harness cannot, from the outside, tell whether a postflop action came from the ranged heuristic or the distilled net — both are inside `engine.decide()`. Postflop leaks are labeled `postflop-heuristic` by default; isolating net-vs-heuristic attribution would require instrumenting the engine (out of scope — other agents own `src/`).
